# evals/ — 通用校準題組

## 本文件目的

本目錄提供六種任務原型各 3 題(基礎/應用/陷阱)共 18 題的校準題組,用於定期抽測執行模型(如 Opus):量測它在八維度(R1-R8)上的實際水準,找出低分維度,回頭補對應的示範與規則。陷阱題取材自 ANTIPATTERNS.md,專測「會不會掉進已知反模式」。

## 如何使用本文件

### 施測方式

1. **抽題**:每輪從 2-3 個原型各抽 1 題;同一題不連續兩輪使用(受測模型可能殘留記憶)。
2. **乾淨對話施測**:開全新對話,只貼題目,不貼任何 harness 協議 → 得到**基線分數**。
3. **加載 harness 再測**:另開新對話,依總 README 的載入順序貼入協議與對應示範,再貼同一題 → 得到 **harness 增益分數**。兩者之差就是這套框架對該模型的實際貢獻。
4. **判卷**:把「題目 + 受測模型的作答」貼給任一模型,使用 `GENERAL_RUBRIC.md` 的〈Grader Prompt〉(自包含,可直接複製)評分;再對照該題「評分說明」中的鑑別維度與自動扣分項覆核。每題的「標準答案」小節供判卷者比對關鍵要點。

### 施測頻率

建議:每月一輪;或在下列事件後加測一輪——更換執行模型版本、harness 協議文件更新、實際任務中發現新的失手模式(此時優先抽對應原型的陷阱題)。

### 低分維度的回饋迴路

哪個維度低,就回頭補哪類材料:

| 低分維度 | 回補動作 |
|---|---|
| R1 正確性 | 檢查是否幻覺 API/數據 → 在 ANTIPATTERNS.md 對應原型節強化「查證後才寫」條目;施測時把對應 EXEMPLAR 的【執行】段納入 few-shot |
| R2 完整性 / R4 邊界處理 | 補貼對應 EXEMPLAR 的【驗收定義】與邊界清單段落;在專案 CHECKLIST 加入邊界情境清單欄 |
| R3 範圍紀律 | 補貼 EXEMPLAR 的【範圍】段(目標/非目標);要求作答前先產出非目標清單 |
| R5 可驗證性 | 要求每個結論附驗證途徑;補貼 GENERAL_RUBRIC R5 錨點與對應 CRITIQUE 批改點 |
| R6 簡潔性 / R7 可維護性 | 補貼對應 CRITIQUE.md 的過度工程批改點;施測指令中明訂規模上限 |
| R8 誠實度 | 補貼 DECISION_PROTOCOL.md 第 4 節(不確定性表達規範)全文;判卷時逐字檢查信心標註 |

連續兩輪同維度低分 → 不只補材料,考慮把該維度的要求寫進 CLAUDE.md 固定工作流的交付聲明格式。

### 紀錄格式建議

每輪施測留一列紀錄,累積成趨勢:

```markdown
| 日期 | 模型版本 | 題目 | 施測方式(基線/harness) | R1-R8 分數 | 最低維度 | gap 與回補動作 |
|---|---|---|---|---|---|---|
| 2026-07-01 | (版本) | DB-E3 | harness | 5/4/3/4/4/4/4/2 | R8 | 改測試遷就錯誤未揭露 → 補貼 DECISION_PROTOCOL 第4節 |
```

## 18 題一覽

| 原型 | 基礎 | 應用 | 陷阱 |
|---|---|---|---|
| 軟體實作(`software-implementation.md`) | 時間長度字串解析器 | CSV 使用者批次匯入器 | 為既有客戶端加上重試與批次查詢 |
| 系統設計(`system-design.md`) | SD-E1 API 速率限制元件 | SD-E2 為既有單體 CRM 加全文搜尋 | SD-E3 內部簽核系統「雲原生升級」 |
| 研究分析(`research-analysis.md`) | RA-E1 四天工作週證據綜合 | RA-E2 AI 程式碼助理導入評估 | RA-E3 董事會前的「雲端大回遷」備忘錄 |
| 技術寫作(`technical-writing.md`) | EV-TW-01 虛構 CLI 工具快速上手指南 | EV-TW-02 混亂內部筆記改寫成值班 Runbook | EV-TW-03 更新設定參考文件 |
| 除錯排錯(`debugging.md`) | DB-E1 CSV 報表腳本 TypeError | DB-E2 只在 CI 出現的間歇性測試失敗 | DB-E3 「測試太嚴」的金額測試 |
| 資料分析(`data-analysis.md`) | DA-E1 客服處理時間的典型值 | DA-E2 併購後的合併營收成長率 | DA-E3 訓練營成效的董事會簡報 |

每題四小節:題目(自包含、可直接貼)/出題意圖(陷阱題註明針對的 AP ID)/標準答案(可據以判卷的要點)/評分說明(鑑別維度的 5/3/1 樣貌+自動扣分項)。

## 注意事項

- 題內數據、程式碼、日誌皆為虛構教學材料,題目中已標明;受測模型指出材料為虛構不影響評分。
- 分數不用於跨模型排名(不同題錨點難度不同),只用於同一模型的縱向趨勢與維度診斷。
- 題目一旦懷疑已進入受測模型的訓練資料或被大量引用,參照既有題型換題(保持「基礎/應用/陷阱」結構與 AP 對應關係)。
