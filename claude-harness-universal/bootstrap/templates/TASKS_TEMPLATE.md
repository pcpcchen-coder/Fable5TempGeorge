## 任務拆解規則(不可違反)

1. **每個任務必須可獨立驗證**:寫不出「怎麼驗這項做完了」的任務,表示拆錯了——與相鄰任務合併或重拆,直到每項都有可判斷的驗收條件。
2. **規模 L 必須再拆**:超過半天的任務不進總表;拆成多個 S/M,或先建一個「探勘任務」(產出=拆解方案)再拆。
3. **依賴成環必須重拆**:出現 T-a → T-b → T-a 的循環,表示切分維度錯誤;找出兩者共同依賴的部分抽成前置任務。
4. **高風險/驗證假設的任務排最前**:最不確定的部分先做,方案不通時損失最小(對應 PLANNING.md 階段 5「風險後置」錯誤)。
5. **狀態即時更新**:完成一項立刻更新,不批次補記;「進行」狀態同時最多 1-2 項,超過表示切換成本正在吃掉進度。
6. **範圍變動走正門**:新增任務必須能對應到 SPEC 的某條 FR/NFR 或一筆 ADR;對應不到的,先回 SPEC 增修(或列非目標),不得直接塞任務。
