# BOOTSTRAP_PROMPT — 新專案啟動指令

## 本文件目的

本文件是一段可直接貼給執行模型(如 Opus)的啟動指令。目的:讓任何新專案的「規劃品質」由協議與模板保證,而不是依賴模型即興發揮。使用者只需提供專案描述,執行模型依指令走完規劃階段、生成整組專案文件,**經使用者確認後才進入實作**。

## 如何使用本文件

1. 開一個新對話,先貼上(或掛載)claude-harness-universal 套件:至少包含 `PLANNING.md`、`DECISION_PROTOCOL.md`、`GENERAL_RUBRIC.md`、`ANTIPATTERNS.md` 與 `bootstrap/` 目錄。
2. 貼上下方「啟動指令」整段。
3. 接著貼上你的專案描述(想做什麼、給誰用、已知限制)。
4. 依執行模型的節奏逐階段確認;全部確認前它不會寫任何實作程式碼。

---

## 啟動指令(整段複製,貼上後接著貼專案描述)

```text
你將協助啟動一個新專案。請嚴格依下列程序執行;在第 6 步「使用者確認」完成之前,
禁止撰寫任何實作程式碼或實作性內容——你現在的唯一交付物是規劃文件組。

【第 1 步:讀取與澄清】
讀取我接下來提供的專案描述,然後依 PLANNING.md 的階段 1「需求澄清」執行:
- 對照「必須先問使用者」清單,把命中的問題一次問完(編號列出、附選項與建議預設、
  說明為何而問);
- 可假設的項目用標準格式寫出:「假設:…;若不符請告知,影響範圍是…」;
- CLAUDE.md.template 中填不出來的專案變數({{技術棧}}、{{測試指令}} 等),
  一併列入這次提問,不可猜填。
等我回覆後才進入第 2 步。

【第 2 步:規劃階段】
依 PLANNING.md 依序走完:問題重述 → 範圍界定 → 方案枚舉與取捨 → 分解 → 定義驗收。
每個階段的產出按 PLANNING.md 該階段的「產出格式」寫出,並通過其「品質門檻」自查。
過程中的方案選擇依 DECISION_PROTOCOL.md 第 1 節的優先序裁決並留下紀錄。

【第 3 步:生成專案文件組】
用 bootstrap/templates/ 的模板,把第 2 步的規劃產出固化成本專案的檔案:
- SPEC.md(用 SPEC_TEMPLATE.md;目標/非目標/FR/NFR/假設/開放問題/驗收標準)
- tasks.md(用 TASKS_TEMPLATE.md;每項任務含依賴、規模、風險級、驗收條件)
- adr/ADR-001-*.md(用 ADR_TEMPLATE.md;記錄啟動期命中 ADR 觸發清單的決定,
  至少含技術棧選擇)
- CHECKLIST.md(用 CHECKLIST_TEMPLATE.md,代入本專案名稱)
- verify.sh(用 templates/verify.sh 骨架,填入本專案的 lint/test/build 指令;
  暫時沒有的區段保留 SKIP 並註明何時補上)
- CLAUDE.md(用 CLAUDE.md.template,把所有 {{變數}} 填成本專案的值;
  固定工作流部分一字不改)
模板中的「填寫指引」註解在填完後刪除;任何欄位填不出來,列入「待確認問題」,
不可留空或猜填。

【第 4 步:自審】
依 GENERAL_RUBRIC.md 的〈交付前自評指令〉對整組文件自評(R1-R8,附證據);
任一維度低於 3 分先修正再交付。同時檢查:SPEC 的每條目標都有對應驗收標準、
tasks.md 無 L 級任務與循環依賴、CLAUDE.md 無殘留的 {{變數}}。

【第 5 步:分批提交確認】
把產出分兩批給我過目,節奏如下:
- 第一批:SPEC.md + 開放問題清單(這決定方向,先確認);
- 第二批:tasks.md + ADR + CHECKLIST.md + verify.sh + CLAUDE.md(方向確認後的落地)。
每批附一段 5 行以內的摘要:關鍵決定、主要取捨、待我拍板的事項。

【第 6 步:等待確認】
我對兩批文件都明確表示同意後,你才可以開始實作,且實作必須依 CLAUDE.md 的
固定工作流執行。我若提出修改,更新對應文件並重走受影響部分的自審,再次提交。
在取得同意前,你只能回答問題與修改規劃文件。
```

## 啟動後的檢查點(給使用者)

收到執行模型的產出時,快速檢查以下五點,任何一點不符就退回:

1. 它有沒有先問問題就直接生成整組文件?(專案描述幾乎不可能完整到不用澄清)
2. SPEC.md 的非目標清單是否為空?(空=範圍界定沒做)
3. tasks.md 是否有寫不出驗收條件的任務?
4. CLAUDE.md 是否殘留未填的 `{{變數}}` 或被改動的固定工作流?
5. 它有沒有在你同意前就開始寫實作程式碼?
