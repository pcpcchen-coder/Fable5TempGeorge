# 劣質解剖析:資料分析(data-analysis)

## 本文件目的

本文件呈現與 `EXEMPLAR.md` 同題(結帳流程 A/B 測試是否全量上線)的一份典型劣質解,並逐點批改。劣質解模擬執行模型(如 Opus)在沒有協議約束時的真實輸出:計算步驟都「算對了」、行文自信流暢,卻因為從未檢查比較本身的有效性,整份報告建立在失真數字上。目的是讓執行模型學會「錯的長什麼樣」,在自審時能對號入座。

## 如何使用本文件

1. 先讀 `EXEMPLAR.md` 再讀本文件;批改點的「正確做法」欄多處指回黃金示範的具體段落。
2. 注意劣質解最危險的特性:**它的算術幾乎全對**(z 值、p 值、營收乘法都驗算得過),錯的是「算什麼」而不是「怎麼算」。資料分析的審查必須先審比較的有效性,再審計算。
3. 自審 DA 類產出時,把第二部分的批改表當作額外檢查清單:自己的產出若有任何一句與「位置(引述)」欄的句式相似,即為警訊。
4. AP ID 對照 `ANTIPATTERNS.md` 資料分析(DA)節;Rubric 維度對照 `GENERAL_RUBRIC.md`。

## 第一部分:典型劣質解(全文照錄)

以下是劣質解全文(數據承接同一委託,均為教學用虛構)。閱讀時先自己找問題,再對照第二部分。

```markdown
# 結帳流程 A/B 測試分析報告

## 一、測試結果總覽

本次 A/B 測試於 6/8 至 6/21 進行,共納入 40,000 名使用者,資料完整無缺
失,樣本量遠超統計檢定所需,可直接進行分析。

| 組別 | 轉換率 |
|---|---|
| 舊版 | 4.04% |
| 新版 | 4.99% |

新版轉換率相對提升 23.7%。經雙比例 z 檢定,z = 4.31,p < 0.0001,遠低
於 0.05 的顯著水準,代表我們有超過 99.99% 的把握確定新版優於舊版(已
用線上統計計算機交叉驗證)。A/B 測試採隨機分組,兩組使用者特徵完全一
致,因此不需要額外檢查干擾因素,結論在統計上非常穩固。

## 二、分段表現:全面提升

| 分段 | 舊版轉換率 | 新版轉換率 | 相對提升 |
|---|---|---|---|
| iOS | 4.50% | 5.50% | +22.2% |
| Android | 3.80% | 4.70% | +23.7% |
| Web | 3.36% | 4.42% | +31.6% |
| 新客 | 2.81% | 3.42% | +21.6% |
| 回頭客 | 4.56% | 5.67% | +24.2% |

所有分段全面提升,結論非常穩健。其中 Web 平台提升最大(+31.6%),顯示
新流程在桌面情境下優勢更明顯;新客提升 21.6%,說明新流程對第一次購買
的使用者特別友善。簡報建議重點呈現 Web 與新客這兩個亮點。

圖表建議:轉換率長條圖的 Y 軸從 3.5% 開始,讓兩組的差距更一目了然。

## 三、為什麼新版更好

新版把結帳從 5 步縮減為 3 步。根據 Baymard Institute 的研究,結帳步驟
每減少一步,轉換率平均可提升約 10%;本次減少 2 步,理論提升約 20%,與
實測的 23.7% 高度吻合,交叉印證了測試結果的可靠性。此外,業界平均結帳
轉換率約為 2.3%,我們兩組均遠高於業界水準,反映整體結帳體驗的優勢。

## 四、商業影響估算

以每月約 80,000 名使用者進入結帳頁、業界平均客單價 NT$1,200 估算:

- 現有月轉換:80,000 × 4.04% ≈ 3,232 筆
- 上線後增量:3,232 × 23.7% ≈ 766 筆
- 每月新增營收:766 × NT$1,200 ≈ NT$919,000

全量上線後預計每月可新增營收約 92 萬元,一年約 1,100 萬元。

## 五、風險評估

有人可能會擔心新鮮感效應(novelty effect),但本次 p < 0.0001 的顯著
性極強,即使存在部分新鮮感成分,也不可能解釋全部提升;且測試已涵蓋完
整兩週,足以排除短期波動的影響。整體而言,本次上線風險很低。

## 六、結論與建議

1. 建議下週產品會議直接宣布全量上線,數據面沒有問題。
2. Web 平台提升最大,建議優先將新版的設計語言推廣到商品頁與購物車頁。
3. 建議行銷部門以「結帳體驗升級,轉換率提升 23%」作為對外宣傳點。
4. 上線後持續觀察數據即可。
```

### 改寫示範:三個代表性病句的前後對照

改寫的共同原則:先修比較的有效性,再談數字;信心與證據對齊;每個數字可重算。

**改寫 1(批改 2,失真的頭條數字):**

```text
原句:新版轉換率相對提升 23.7%……p < 0.0001,結論在統計上非常穩固。
改寫:合併兩週的 +23.7% 不可使用:分流比例曾從 10% 調至 50%,且第二週
      與年中慶重疊,新版樣本 83% 集中在高轉換週(加權失真)。在唯一可
      比的第二週(50/50),提升為 +3.0%(515/10,000 對 500/10,000),
      95% CI [-9.2%, +15.2%],p = 0.63——現有數據無法判定新版更好或更差。
```

**改寫 2(批改 10,用顯著性關閉新奇效應):**

```text
原句:p < 0.0001 的顯著性極強,即使存在部分新鮮感成分,也不可能解釋全
      部提升。
改寫:相對提升隨時間單調遞減(週 1 +20.0% → 週 2 前半 +5.6% → 週 2 後
      半 +0.8%),形狀與新奇效應一致;但各段皆不顯著,新奇效應、促銷交
      互與小樣本波動三種解釋無法在本數據中區分(信心:低)。可確定的
      是:+20% 不可作為長期效果的預期值。
```

**改寫 3(批改 12,空話式監控):**

```text
原句:上線後持續觀察數據即可。
改寫:若因非數據理由先上線,保留 10% 舊版對照組 4 週:當新版相對變化
      的 95% CI 上界 < 0,或退款率、結帳耗時任一護欄指標顯著惡化,即回
      滾舊版;護欄數據申請到位是上線的前置條件。
```

### 這份劣質解最致命的三個問題

1. **一個沒問出口的問題毀掉整份報告(批改 1、2;AP-DA-01)**:「為什麼兩組是 28,000 對 12,000?」——只要問了這一句,分流變動與促銷重疊就會浮出,+23.7% 就會當場作廢。劣質解的所有後續計算都正確地建立在這個沒被檢查的地基上;資料分析的第一課是先驗資料與設計,再碰統計量。
2. **把 p 值當成萬用免死金牌(批改 2、3、4、10)**:p < 0.0001 在文中被用來證明「新版更好」「99.99% 確定」「不需查干擾」「新奇效應無關」——但 p 值只回答一個窄問題(在此比較下,差異是否可能純屬機遇),不回答「比較本身是否有效、效果有多大、能否持續」。顯著性無法拯救錯誤的比較,更不能替代偏誤檢查。
3. **不確定性歸零且不可重現(批改 3、5、7、11;AP-DA-04、AP-DA-09、AP-DA-10、AP-DA-11)**:全篇無信賴區間、無樣本數、無腳本,配上捏造的業界法則與截斷 Y 軸的建議——讀者被剝奪了每一個檢查的把手,只能選擇相信;而最需要被檢查的那個數字,恰好就是錯的。
