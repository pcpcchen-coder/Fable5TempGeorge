# 黃金示範:資料分析(data-analysis)

## 本文件目的

本文件是資料分析(DA)原型的完整黃金示範:以一個真實委託(A/B 測試是否全量上線)示範八階段規劃協議的完整走法,思考全程外顯。題目中埋有兩個真實世界常見的陷阱——新奇效應與分段悖論——示範如何靠固定流程(先體檢、再檢定、顯著性與效果量並重、誠實標註不確定性)把它們挖出來,而不是靠靈感。

## 如何使用本文件

1. 接到 DA 類任務時,先完整讀過本示範一次,再開始自己的【澄清】。
2. 八個思考段落標籤(【澄清】【重述】【範圍】【方案取捨】【分解】【驗收定義】【執行】【自審】)的順序與格式照抄,內容換成當前任務。
3. 「◆ 為什麼這樣做」是寫給你看的後設註解,說明該步在防哪個反模式(AP ID 見 `ANTIPATTERNS.md`);你自己的產出中不需要輸出這些註解,但必須做到它們指出的事。
4. 報告中每個數字都必須能對應到附錄腳本的一行輸出——這是 DA 原型的硬規則(AP-DA-11),不得省略。

> **數據聲明**:本示範中所有數據均為教學用虛構數據(依 AP-DA-04 的規範明確標註),但全部數字內部一致,可由附錄腳本重算驗證。

## 【澄清】

任務描述有兩個立即可見的警訊:(1) 兩組樣本數是 28,000 對 12,000,不是常見的 50/50——分流比例的歷史沒有交代;(2) 委託的問法是「幫我看一下數據沒問題吧」——這是確認式框架,期待的答案已經寫在問題裡,分析者必須把它翻譯回開放問題。逐句檢查後,命中 PLANNING.md 表 A 的兩種「必須先問」情況:#6(需要使用者才有的資訊:實驗設計、分流歷史、指標口徑、同期活動)與 #3(缺少驗收標準:「沒問題」與「值得上線」的判準是什麼)。依規則一次問完:

```markdown
開工前需要確認 5 件事(可只回選項字母或簡答):
1. 兩組樣本為何是 28,000 對 12,000?分流比例在實驗期間有沒有變動過?
   (為何而問:分流比例若曾變動,直接合併兩週數據會產生加權失真,
    這是彙總比較最常見的失效原因;答案決定你的 +23% 能不能用。)
2. 「轉換率」的口徑:分子與分母各是什麼?去重與歸因窗怎麼定?兩組是否相同?
   (為何而問:口徑不明的比較無法覆核;分子分母差一個定義,結論可以差一倍。)
3. 實驗期間(6/8–6/21)有沒有促銷、行銷活動或改版等同期事件?
   (為何而問:同期事件若與分流變動重疊,會系統性扭曲合併結果。)
4. 可否提供按週(或按日)拆分的分組數據?
   (為何而問:判斷新奇效應與時間混淆的唯一途徑是看效果隨時間的變化。)
5. 上線的決策標準是什麼?提升多少才值得上?有沒有護欄指標(客單價、退款率)?
   A. 有正式門檻(請提供) B. 沒有,想聽建議 —— 建議:照實回答
   (為何而問:「該不該上線」必須對照事先定義的門檻,否則分析只能回答
    「有沒有差異」而回答不了「該不該做」。)
```

假設:

提問:
- 已於上方一次問完,無遺留問題;護欄指標無數據,列入非目標與限制。

◆ 為什麼這樣做:第 1、3、4 題是這個任務的生死題——不問分流歷史與同期事件,後面的一切計算都會建立在失真的合併數據上。這是 AP-DA-01(不驗資料先分析)在 A/B 測試情境的具體形態;第 2 題防 AP-DA-08(口徑不一致):口徑沒寫清楚之前,連「4.99%」這個數字本身是什麼意思都不知道。

## 【重述】

問題:PM 要的表面上是「幫我確認數據沒問題」,實際目的是「做對全量上線的決策」。兩者可能衝突:如果數據其實不支持上線,委託的正確產出是說服力足夠的「不建議」,而不是替既定結論背書。手段(檢查數據)服務於目的(決策正確),不是服務於「讓簡報好看」。
原型:data-analysis(DA)。
可能誤解點:
1. 把任務當成「驗算 PM 的 p 值」——p 值算對了也可能整個比較就是錯的,要審的是比較本身的有效性。
2. 「分段數據整理成簡報結論」不等於「照 PM 期待的方向整理」——若分段結論不成立,如實告知也是交付的一部分。
3. 「沒問題吧」的社交壓力可能誘使分析淡化壞消息——結論方向必須只由數據決定。

## 【範圍】

目標:
1. 判定現有數據是否支持「全量上線」,給出條件式決策建議(建議動作+成立條件+重評觸發)。
2. 對 PM 版分析(合併 +23.7%、分段全面提升)給出明確的「可用/不可用」判定與原因。
3. 全部數字可由附帶腳本重算;報告中每個轉換率旁附樣本數與轉換數。

非目標:
1. 不分析客單價、退款率等護欄指標——本次無數據;列入決策建議的前置條件。
2. 不重新設計實驗平台或分流系統——僅就本次數據給實驗設計的改進建議。
3. 不製作簡報圖表——交付數據表格與可直接引用的結論句;若後續做圖,須遵守 AP-DA-10/AP-DA-12 的規範。
4. 不建預測模型或使用者行為歸因——超出「該不該上線」所需。

邊界裁定:PM 只要求「看數據」,但「數據能不能用」取決於實驗設計(分流歷史、同期事件),故設計層面的檢查納入範圍——它是資料品質檢查的構成部分,不是範圍蔓延。

## 【方案取捨】

三個候選的分析路線:

方案 A:直接檢定 PM 的合併數據(雙比例 z 檢定,28,000 對 12,000)。優點:最快,十分鐘出結果。缺點:已確認分流比例跨週變動(10% → 50%)且第二週與大促重疊——合併比較讓新版樣本集中在高轉換週,是**已知會產生錯誤結果**的情境。成本:極小。

方案 B:限定第二週(50/50 平穩期)做主分析,第一週與時間趨勢作為新奇效應的輔助證據,分段全部重算在第二週子集上。優點:兩組在同一時間窗、同一分流比例下直接可比,計算透明,PM 可自行驗算。缺點:放棄第一週樣本,檢定力下降(須誠實揭露)。成本:小。

方案 C:用統計模型合併兩週(分層 Mantel-Haenszel 加權,或邏輯斯迴歸加入 組別×週次 交互項)。優點:用到全部資料,方法學上站得住。缺點:合併估計的對象是「兩週期間的平均效果」,其中混入第一週的新奇效應與放量期異常——這不是上線決策要的量(長期效果);且對非統計背景的會議聽眾可驗證性差。成本:中。

比較維度:正確性(估計對象是否為決策所需的量)、簡單性、可驗證性。
選定:B。理由:唯一同時滿足「比較有效」與「估的是決策相關量」的路線;檢定力損失以誠實揭露和「延長實驗」的建議補償。
否決 A:依 DECISION_PROTOCOL 準則 1(正確性)淘汰——存在已知會產生錯誤結果的情境(加權失真),正確性永遠不可被速度交換,無例外。
否決 C:準則 1 存疑(估計對象錯配:期間平均混入新奇效應,不等於長期效果)、準則 3(簡單性)落敗——在 B 已能回答決策問題時,C 增加模型假設與出錯環節,卻不增加決策資訊;留作附註供統計背景讀者參考。

◆ 為什麼這樣做:方案 A 正是 PM 已經做的事,也是劣質解的必經之路——它不是「比較粗糙」,而是在準則 1 就該被淘汰的錯誤比較。把它列為正式候選並寫明否決理由,是為了讓報告讀者(和未來的自己)看見這條路為什麼不能走。

## 【分解】

1. 資料體檢:加總一致性(三種切分的合計=總體)、口徑覆核、SRM 檢定(各週觀察分流 vs 設計分流)、兩組組成平衡(高風險:任何一項失敗,結論轉為「實驗數據無效,需重跑」,後續各項全部中止)。完成定義:體檢表全項有數字與判定。依賴:無。
2. 重建時間結構:按週拆分,確認分流變動與促銷的重疊,判定可比時段。完成定義:週表完成,合併失真的機制寫清楚。依賴:1。
3. 主分析:第二週雙比例 z 檢定、95% CI、效果量、檢定力/MDE。完成定義:四個數字齊備且腳本可重算。依賴:2。
4. 新奇效應檢查:效果隨時間的變化(週間與週內),誠實標註可分離與不可分離的解釋。完成定義:趨勢表+信心標註。依賴:2。
5. 分段重析:五個分段全部改在第二週子集重算,與 PM 版對照,標註多重比較。完成定義:分段表+比較次數揭露。依賴:3。
6. 撰寫報告與可重跑腳本,執行腳本核對全部數字,自審。完成定義:腳本實跑輸出與報告一致。依賴:3、4、5。

◆ 為什麼這樣做:體檢排第一且設為「失敗即全停」——AP-DA-01(不驗資料先分析)是阻斷級反模式;若 SRM 失配,後面算得再精緻都是對無效實驗的精緻分析。

### 結帳流程 A/B 測試覆核報告(數據為教學用虛構)

#### 0. 結論摘要

**不建議在下週會議上以現有數據宣布全量上線。**「提升 23.7%」是分流比例變動(第一週 10%、第二週 50%)與年中慶大促重疊造成的加權失真,不是新版的真實效果;在唯一可比的第二週(50/50 平穩期),新版相對提升 +3.0%,95% CI [-9.2%, +15.2%],統計上不顯著,且效果隨時間遞減的模式與新奇效應一致。數據同樣**不支持**「新版更差」——正確的下一步是用乾淨的設計把效果量測出來(選項見第 6 節),而不是在失真數據上二選一。

#### 1. 資料體檢與實驗有效性

| 檢查項 | 結果 | 判定 |
|---|---|---|
| 加總一致性 | 平台、新舊客、週次三種切分的合計皆等於總體(腳本 assert 全數通過) | 通過 |
| 口徑 | 分子=觸發分組後 24h 內完成付款之去重使用者;分母=進入結帳頁之去重使用者;兩組相同。注意:單位為「使用者-週」,跨週重複活躍者(PM 估約 15%,未驗證)兩週各計一次 | 通過,附註 |
| SRM 檢定(週 1) | 觀察新版佔比 2,000/20,000 = 10.0% vs 設計 10%,chi2 ≈ 0,p ≈ 1.0 | 通過 |
| SRM 檢定(週 2) | 觀察 10,000/20,000 = 50.0% vs 設計 50%,chi2 ≈ 0,p ≈ 1.0 | 通過 |
| 組成平衡(週 2) | 新客佔比:舊版 30.0% vs 新版 30.0%;iOS 佔比 40.0% vs 41.0%(p = 0.15) | 通過 |
| 列級品質 | 僅有彙總數據,缺值/重複列/異常值無法檢查;bot 過濾規則兩組相同(PM 確認,未稽核) | 有限通過,列入限制 |

SRM(sample ratio mismatch,樣本比例失配)= 觀察到的分流比例偏離設計值;失配代表分流或記錄有問題,實驗整體作廢。本次兩週各自通過——**實驗內部有效,失真發生在「跨週合併」這一步**,見下節。

#### 2. 為什麼「+23.7%」不能用

按週拆分(每格為 轉換數/使用者數 = 轉換率):

| 期間 | 新版分流 | 舊版 | 新版 | 相對提升 | p 值 |
|---|---|---|---|---|---|
| 週 1(6/8–6/14,放量期) | 10% | 630/18,000 = 3.50% | 84/2,000 = 4.20% | +20.0% | 0.11 |
| 週 2(6/15–6/21,年中慶) | 50% | 500/10,000 = 5.00% | 515/10,000 = 5.15% | +3.0% | 0.63 |
| 合併(PM 版) | — | 1,130/28,000 = 4.04% | 599/12,000 = 4.99% | **+23.7%** | <0.0001 |

合併後的 +23.7% 比任何一週的提升(+20.0%、+3.0%)都大——正確加權下這不可能發生,這是辛普森悖論(Simpson's paradox)的合併變形。機制:年中慶使第二週兩組的轉換率整體墊高(5.0% vs 3.5%),而新版樣本有 83%(10,000/12,000)落在這個高轉換週,舊版只有 36%(10,000/28,000);合併等於拿「主要在促銷週購物的新版使用者」對比「主要在平常週購物的舊版使用者」。PM 算的 p < 0.0001 是真的——但它檢定的是這個失真後的差;**顯著性無法拯救一個錯誤的比較**。

#### 3. 主分析(週 2,50/50 平穩期)

選第二週為主分析的理由:(a) 兩組同時間窗、同分流比例,直接可比;(b) 樣本量是週 1 新版組的 5 倍;(c) 距首次曝光最遠,最接近長期效果。

- 舊版 500/10,000 = 5.00%,新版 515/10,000 = 5.15%。
- 絕對差 +0.15 個百分點;相對提升 +3.0%。
- 雙比例 z 檢定:z = 0.48,**p = 0.63**;差值 95% CI [-0.46pp, +0.76pp],相對 [-9.2%, +15.2%]。
- 檢定力:每組 n = 10,000、基準 5.0% 下,本實驗只偵測得到相對提升約 17% 以上的效果(α=0.05 雙尾、power 0.80)。換句話說,即使新版真有 +5% 的效果,本設計也大概率看不出顯著。

「統計上不顯著」不等於「效果為零」:CI 告訴我們,數據與「-9.2% 到 +15.2% 之間的任何真實效果」都相容——包括 0,也包括值得上線的 +5%。**現有數據回答不了這個決策,這本身就是本次分析最重要的發現。**

#### 4. 新奇效應檢查

效果隨時間的變化(相對提升,各段皆為獨立比較):

| 時段 | 舊版 | 新版 | 相對提升 |
|---|---|---|---|
| 週 1(6/8–6/14) | 630/18,000 = 3.50% | 84/2,000 = 4.20% | +20.0% |
| 週 2 前半(6/15–6/17) | 234/4,500 = 5.20% | 246/4,480 = 5.49% | +5.6% |
| 週 2 後半(6/18–6/21) | 266/5,500 = 4.84% | 269/5,520 = 4.87% | +0.8% |

點估計單調遞減(+20.0% → +5.6% → +0.8%),是新奇效應(novelty effect,使用者對新介面的嘗鮮反應隨曝光衰減)的典型形狀。但誠實地說,三段各自的 CI 都跨 0 且彼此重疊,以下三種解釋在本數據中**無法區分**:(a) 新奇效應衰減;(b) 促銷週高意圖買家對結帳流程差異不敏感;(c) 週 1 新版僅 2,000 人(84 次轉換)的隨機波動。三種解釋指向同一個結論:**+20% 與 +23.7% 都不可作為長期效果的預期值**(信心:高);衰減是否存在、長期效果落在哪裡,需要更長的乾淨觀測(信心:低,無法以現有數據裁決)。

#### 5. 分段重析(週 2 子集;探索性,共 5 個比較,未做多重比較校正)

| 分段 | 舊版 | 新版 | 相對提升 | p 值 | PM 合併版 |
|---|---|---|---|---|---|
| iOS | 220/4,000 = 5.50% | 242/4,100 = 5.90% | +7.3% | 0.43 | +22.2% |
| Android | 235/5,000 = 4.70% | 228/4,900 = 4.65% | **-1.0%** | 0.91 | +23.7% |
| Web | 45/1,000 = 4.50% | 45/1,000 = 4.50% | 0.0% | 1.00 | +31.6% |
| 新客 | 96/3,000 = 3.20% | 105/3,000 = 3.50% | +9.4% | 0.52 | +21.6% |
| 回頭客 | 404/7,000 = 5.77% | 410/7,000 = 5.86% | +1.5% | 0.83 | +24.2% |

三個發現:
1. **PM 版分段全面失真**:合併失真滲入每一個分段(每個分段內同樣是新版樣本集中在促銷週),Android 從 +23.7% 翻轉為 -1.0%,「最亮眼」的 Web(+31.6%)歸零——且 Web 全段只有 45+45 次轉換,本來就不足以單獨下結論。
2. 修正後**無任何分段顯著**;五個比較未校正,即使有一段 p < 0.05 也應視為探索性訊號。
3. 新客 +9.4% 是唯一值得記下的方向假說(新客沒看過舊版,不受新奇效應影響,若為真值得重視)——但 p = 0.52,只能作為下一輪實驗的觀察重點,不能進簡報結論。

**給 PM 的直白版**:你想放進簡報的「新客與 Web 提升很棒」在修正後不成立;能誠實放進簡報的分段結論只有一句——「各分段均無可靠差異,新客方向值得下一輪驗證」。

```python
#!/usr/bin/env python3
# 結帳流程 A/B 測試分析腳本(教學用虛構數據)。執行:python3 ab_analysis.py
import math

def norm_sf(z):  # 標準常態右尾機率
    return 0.5 * math.erfc(z / math.sqrt(2))

def two_prop_ztest(x1, n1, x2, n2):
    p1, p2 = x1 / n1, x2 / n2
    pooled = (x1 + x2) / (n1 + n2)
    z = (p2 - p1) / math.sqrt(pooled * (1 - pooled) * (1 / n1 + 1 / n2))
    se = math.sqrt(p1 * (1 - p1) / n1 + p2 * (1 - p2) / n2)
    return p1, p2, z, 2 * norm_sf(abs(z)), (p2 - p1) - 1.96 * se, (p2 - p1) + 1.96 * se

def srm_chi2(obs_t, obs_c, ratio_t):  # 觀察分流 vs 設計分流
    n = obs_t + obs_c
    et, ec = n * ratio_t, n * (1 - ratio_t)
    chi2 = (obs_t - et) ** 2 / et + (obs_c - ec) ** 2 / ec
    return chi2, 2 * norm_sf(math.sqrt(chi2))

def mde_rel(p, n):  # alpha=0.05 雙尾、power=0.80 的近似相對 MDE
    return (1.96 + 0.84) * math.sqrt(2 * p * (1 - p) / n) / p

def report(title, x1, n1, x2, n2):
    p1, p2, z, pv, lo, hi = two_prop_ztest(x1, n1, x2, n2)
    print(f"[{title}] 舊 {x1}/{n1}={p1:.4%} 新 {x2}/{n2}={p2:.4%} "
          f"相對 {p2/p1-1:+.2%} z={z:.3f} p={pv:.4g} "
          f"CI[{lo:+.4%},{hi:+.4%}] 相對CI[{lo/p1:+.1%},{hi/p1:+.1%}]")

# 1. 體檢:加總一致性(任何一條失敗即丟例外中止)
assert 630 + 500 == 1130 and 18000 + 10000 == 28000        # 舊版 週1+週2=總體
assert 84 + 515 == 599 and 2000 + 10000 == 12000           # 新版 週1+週2=總體
assert 504 + 532 + 94 == 1130 and 264 + 282 + 53 == 599    # 平台合計=總體
assert 236 + 894 == 1130 and 123 + 476 == 599              # 新舊客合計=總體
assert 234 + 266 == 500 and 246 + 269 == 515               # 週2前後半合計=週2
assert 220 + 235 + 45 == 500 and 242 + 228 + 45 == 515     # 週2平台合計=週2
assert 96 + 404 == 500 and 105 + 410 == 515                # 週2新舊客合計=週2
print("體檢:所有分段加總與總體一致")

# 2. SRM
for label, t, c, r in [("週1 設計10%", 2000, 18000, 0.10),
                        ("週2 設計50%", 10000, 10000, 0.50)]:
    chi2, pv = srm_chi2(t, c, r)
    print(f"SRM {label}: chi2={chi2:.3f} p={pv:.3f}")

# 3. 各項比較
report("合併(僅示範其失真)", 1130, 28000, 599, 12000)
report("週1", 630, 18000, 84, 2000)
report("主分析:週2", 500, 10000, 515, 10000)
report("週2前半", 234, 4500, 246, 4480)
report("週2後半", 266, 5500, 269, 5520)
report("週2 iOS", 220, 4000, 242, 4100)
report("週2 Android", 235, 5000, 228, 4900)
report("週2 Web", 45, 1000, 45, 1000)
report("週2 新客", 96, 3000, 105, 3000)
report("週2 回頭客", 404, 7000, 410, 7000)

# 4. 檢定力
print(f"MDE(基準5%, 每組10,000):相對 {mde_rel(0.05, 10000):.1%}")
for w in (3, 4, 6, 12):
    print(f"  再累積 {w} 週(每組 {10000*w:,}):相對 {mde_rel(0.05, 10000*w):.1%}")
```
